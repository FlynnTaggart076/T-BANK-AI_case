# Vacancy Matching Agent

MVP агентного pipeline для поиска стажировок и junior-вакансий. Пользователь пишет запрос в свободной форме, агент извлекает критерии, ищет вакансии через открытые данные Работа России / Trudvsem, валидирует и ранжирует результаты, а затем делает Markdown-отчет с объяснением топ-5.

## Краткое описание

Агент помогает начинающим специалистам быстрее находить junior-вакансии и стажировки без ручного просмотра большого числа карточек. Пользователь пишет запрос обычным языком, а pipeline извлекает критерии, обращается к выбранному источнику вакансий, очищает и дедуплицирует выдачу, считает score и формирует объяснения для топа. LLM используется для извлечения намерения пользователя и для коротких объяснений, а критически важные части - фильтрация, проверка опыта, дедупликация и итоговый score - считаются обычным кодом. Основные ограничения связаны с качеством внешних API: в карточках часто нет структурированных навыков, зарплаты или точного опыта, поэтому часть проверок приходится делать по тексту вакансии. Самым сложным было отделить реальные junior/стажерские позиции от вакансий, где в тексте встречаются senior/middle-маркеры или требования большого опыта. Следующим шагом я бы улучшил обработку зарплатных вилок и добавил больше источников, например HH, чтобы сравнивать выдачу между площадками.

## Быстрый запуск

```powershell
copy .env.example .env
notepad .env
py server.py
```

Открой в браузере:

```text
http://127.0.0.1:8000
```

В форме на сайте выбери источник `Работа России`, `SuperJob` или `Оба`, введи запрос и нажми `Найти вакансии`.

## Stack

- Python 3.11+
- Trudvsem open data API и SuperJob API как источники вакансий
- Локальный `data/sample_vacancies.json` как fallback для демо
- OpenAI LLM через HTTP без обязательных Python-зависимостей
- CLI, Web UI и Telegram-бот используют один агентный pipeline

## Пример входных данных

```text
Ищу junior-вакансию аналитика данных в Москве, можно удаленно, знаю SQL и Python, зарплата от 70000.
```

Для API тот же запрос можно отправить так:

```json

  "top_n": 5,
  "source": "superjob"
}
```

## Run CLI

```powershell
copy .env.example .env
py run.py "Ищу стажировку или junior позицию Python backend, можно удаленно или Москва, FastAPI/PostgreSQL, без опыта или до года."
```

Если в `.env` или переменных окружения задан `OPENAI_API_KEY`, агент использует LLM для извлечения критериев и объяснений. Если ключа нет или модель недоступна, включается эвристический fallback.

Пример `.env`:

```dotenv
OPENAI_API_KEY=<your_openai_key>
OPENAI_MODEL=gpt-4o-mini
SUPERJOB_API_KEY=<your_superjob_key>
TELEGRAM_BOT_TOKEN=<your_telegram_token>
```

Результаты:

- `output/report.md` - итоговый отчет;
- `output/run.log` - trace шагов pipeline.

## Run Web UI

```powershell
py server.py
```

Открой:

```text
http://127.0.0.1:8000
```

Сервер одновременно раздает статический frontend из `frontend/` и принимает API-запрос:

```http
POST /api/search
Content-Type: application/json

{
  "query": "Ищу junior позицию на продуктового аналитика, работа в Москве, без опыта работы",
  "top_n": 5,
  "source": "superjob"
}
```

Поле `source` выбирает инструментарий поиска:

- `trudvsem` - Работа России / Trudvsem;
- `superjob` - SuperJob API, секрет читается из `SUPERJOB_API_KEY`, `SUPERJOB_SECRET`
  или `SUPERJOB_TOKEN` из окружения или `.env`;
- `all` - объединить Работа России и SuperJob перед дедупликацией и scoring.

Ответ совместим с `frontend/app.js`:

## Run Telegram Bot

Токен читается из переменной окружения `TELEGRAM_BOT_TOKEN` или из `.env`.

```powershell
py telegram_bot/bot.py
```

При необходимости можно указать альтернативный env-файл:

```powershell
py telegram_bot/bot.py --env-file /path/to/.env
```

Проверить формат ответа без подключения к Telegram:

```powershell
py telegram_bot/bot.py --dry-run "Ищу junior позицию продуктового аналитика в Москве без опыта" --top 5
```

Команды бота:

- `/start` - краткая инструкция;
- `/help` - подсказка по формату запроса;
- `/top 10 <запрос>` - вернуть от 1 до 25 вакансий;
- обычный текст - поиск топ-5 вакансий.

## Pipeline

1. `extract_criteria` - LLM или эвристика превращает свободный запрос в структуру критериев.
2. `TrudvsemSource` и/или `SuperJobSource` получают вакансии из выбранного источника (`trudvsem`, `superjob`, `all`).
3. `LocalFileSource` добавляет локальные вакансии как fallback для демо, если по основному источнику мало релевантных результатов.
4. `validate_and_dedupe` удаляет битые строки и дубли.
5. `score_vacancies` обычной логикой считает score по роли, стеку, формату, уровню, зарплате, свежести и стоп-словам.
6. `explain_top` через LLM или fallback объясняет топ: совпадения, риски и следующий шаг.
7. `render_report` создает Markdown-отчет.

Отдельный слой `experience_reality_check` проверяет фактический текст карточки и структурные поля вакансии: опыт, квалификацию, требования и обязанности. Если пользователь ищет junior/стажировку/без опыта, вакансии с маркерами `Сеньор`, `Middle`, `Lead`, `Старший`, `от 3 лет`, `2+ лет`, `Опыт работы: 3 лет` жестко уходят в `filtered`, даже если источник вернул их по широкому запросу.

## Integration Points

Основной вход отделен от CLI:

```python
from job_agent import run_agent

result = run_agent(user_query)
```

Для web и Telegram общий JSON-формат собирается в `job_agent.payload`.

## Limitations

- Trudvsem дает меньше серверных фильтров, чем HH/SuperJob, поэтому часть фильтрации делается локально.
- У вакансий часто не хватает структурированного опыта или навыков.
- LLM не является источником истины: финальный score считается обычным кодом, а LLM только извлекает намерение и объясняет результат.
- Для стабильной защиты есть локальный fallback-датасет.
