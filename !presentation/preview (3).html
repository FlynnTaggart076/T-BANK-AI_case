Проблема

Новичку сложно быстро выбрать подходящие вакансии

Что неудобно сейчас

  • Вакансий слишком много.
  • Фильтры на сайтах не всегда точные.
  • Описания нужно читать вручную.
  • Не всегда понятно, подходит ли вакансия по уровню.

Что нужно пользователю

  • Быстро написать запрос своими словами.
  • Получить короткий список вариантов.
  • Понять, почему вакансия подходит.
  • Увидеть риски и следующий шаг.

Работа с данными

Источник вакансий - API “Работа России”

Почему API

  • Данные приходят в JSON.
  • Не нужно парсить HTML-страницы.
  • Проще обрабатывать программно.
  • Можно искать по ключевым словам.
GET /api/v1/vacancies?text=аналитик Из вакансии берем: - название - компанию - город - зарплату - описание - требования - ссылку

Ошибки и ограничения

Что может пойти не так

Обработка ошибок

  • API временно недоступен.
  • Вакансии не найдены.
  • Запрос слишком общий.
  • Данные пришли неполными.

Ограничения

  • Не во всех вакансиях есть зарплата.
  • Описание может быть коротким.
  • AI может ошибиться в критериях.
  • Качество зависит от API.

Итог

Что получилось и как можно развивать проект

Что получилось

  • Сайт принимает запрос пользователя.
  • Агент выделяет критерии.
  • Система получает вакансии через API.
  • Вакансии фильтруются и ранжируются.
  • Пользователь получает топ-5 с объяснением.

Что можно улучшить

  • Добавить hh.ru или SuperJob.
  • Загружать резюме пользователя.
  • Сохранять историю поиска.
  • Добавить авторизацию.
  • Сделать рассылку новых вакансий.

Проект показывает законченный агентный pipeline: входной запрос, AI-анализ, получение внешних данных, обработка и полезный результат для пользователя.
